# 개요

- podman : Pod Manager tool 의 약자

- OCI 표준 Container와 런타임을 개발,관리,실행할 수 있게 해주는 Container engine

- Docker와의 차이점

  - Daemon-less\

    <span class="confluence-embedded-file-wrapper image-center-wrapper"><img src="./media/attachments/356515841/356384921.png" class="confluence-embedded-image image-center" loading="lazy" data-image-src="attachments/356515841/356384921.png" data-height="245" data-width="378" data-unresolved-comment-count="0" data-linked-resource-id="356384921" data-linked-resource-version="1" data-linked-resource-type="attachment" data-linked-resource-default-alias="docker_structure.png" data-base-url="https://never4got10.atlassian.net/wiki" data-linked-resource-content-type="image/png" data-linked-resource-container-id="356515841" data-linked-resource-container-version="2" data-media-id="3346d8a7-f613-4510-b64b-bc9a91795b1b" data-media-type="file" /></span><span class="confluence-embedded-file-wrapper image-center-wrapper"><img src="./media/attachments/356515841/356548618.png" class="confluence-embedded-image image-center" loading="lazy" data-image-src="attachments/356515841/356548618.png" data-height="230" data-width="255" data-unresolved-comment-count="0" data-linked-resource-id="356548618" data-linked-resource-version="1" data-linked-resource-type="attachment" data-linked-resource-default-alias="podman_structure.png" data-base-url="https://never4got10.atlassian.net/wiki" data-linked-resource-content-type="image/png" data-linked-resource-container-id="356515841" data-linked-resource-container-version="2" data-media-id="c94bfdf2-0af5-4be5-a0b1-dda58ef5212a" data-media-type="file" /></span>

    - Docker daemon이 죽으면 모든 컨테이너가 중지

    - podman이 각 컨테이너를 fork/exec 방식으로 실행해서 별도로 구동(daemon 필요 없음)

  - Root 권한 불필요

    - docker daemon이 모든 권한이 집중되다 보니 권한을 제한하기 위해 root 만 docker clinet 제어(docker daemon 를 사용)

    - 시스템 운영자에게 과도한 업무 부하

    - podman은 fork/exec 로 개별 컨테이너 실행(1024 이하의 well known 포트 사용시만 제외하고)

    - 일반 사용자가 실행 가능 → 시스템운영자, 서비스/컨테이너 운영자로 권한 분리 → 보안 측면에서 강점

  - kubernetes 지원

    - podman은 대부분의 명령어와 docker와 호환되지만 docker가 제공되지 않는 기능도 제공

    - kubernetes나 openshift같은 container platform으로 쉽게 이관가능한 kubernetes yaml을 간단히 생성 가능\

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      podman generate kube
      ```

      </div>

      </div>

    - k8s에서 구동중인 pod 디버깅\

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      podman pod
      ```

      </div>

      </div>

# 설치

- Linux container를 구동하기 위해 설계되어 OSX나 Windows에서는 podman 엔진 구동 못함.

- OSX나 windows 에서는 client를 설치하고 원격으로 Linux podman engine을 접속하여 관리

## OSX podman client 설치

- brew 명령어로 설치

- brew install podman

## Windows podman client 설치

- Latest remote client for Windows에서 다운받아 설치

- WSL 2 는 ubuntu linux 이므로 container engine 설치 가능

## Linux

- centos 8 에서는 기본 제공\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  sudo yum -y install podman
  ```

- ubuntu 20.04 이하\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  . /etc/os-release
  echo "deb https://download.opensuse.org/repositories/devel:/kubic:/libcontainers:/stable/xUbuntu_${VERSION_ID}/ /" | sudo tee etc/apt/sources.list.d/devel:kubic:libcontainers:stable.list
  curl -L https://download.opensuse.org/repositories/devel:/kubic:/libcontainers:/stable/xUbuntu_${VERSION_ID}/Release.key | sudo apt-key add -

  sudo apt-get -y upgrade 
  sudo apt-get -y install podman
  ```

## 사용

- podman은 docker와 명령어가 완전히 호환되므로 다음과 같이 alias를 정의해서 사용하면 됨\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  alias docker=podman
  ```
