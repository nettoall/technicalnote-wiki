# 설정

1.  Jenkins관리 \> Manager Users \> admin \> 설정 \> API Token(s)

2.  Add New Token한 후 생성된 token 정보 복사함

3.  build job으로 가서 원격 api호출할 build를 클릭한 후, 구성 클릭 \> “빌드를 원격으로 유발” 체크 후 Authentication Token 이름을 원하는 대로 입력함 ex) build_bms 등

4.  linux telnet 으로 svn 서버 접속 SVN_SERVER/data/repository/업무repository/hooks로 이동한 후, post-commit.tmpl 파일을 post-commit로 rename

5.  commit 즉시 jenkins빌드 수행 스크립트

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    REPOS="$1"
    REV="$2"

    USERNAME=admin
    API_TOKEN=11223kkjds99990. # 2번에서 생성한 token 정보
    AUTH_TOKEN=build_app       # 3번에서 입력한 Authentication Token 이름
    JENKINS_URL=http://localhost:6060/jenkins
    APP_PROJECT_NAME=app_build
    BUILD_URL=$JENKINS_URL/job/$APP_PROJECT_NAME

    if [[ ! -z $BUILD_URL ]];
    then
       BUILD_URL=$BUILD_URL/buildWithParameters?token=$AUTH_TOKEN
       curl -X POST $BUILD_URL --user $USERNAME:$API_TOKEN -v &>/dev/null
       echo "curl -X POST $BUILD_URL --user $USERNAME:$API_TOKEN -v &>/dev/null"
    fi

    echo "[$(date)] Repos : ${REPOS}, Rev : ${REV}" >> ${REPOS}/logs/commit.log
    ```

    </div>

    </div>

6.  테스트 수행. 리눅스 shell 상에서 다음 명령으로 수행

    - ./post-commit "/svn_path/data/repository/app_build" "revision" : 실제 svn에 commit을 하지 않고 가상으로 테스트 가능
