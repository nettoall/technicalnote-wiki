### rpm 명령어

- rpm(Redhat Package Manager)의 약자

- 설치 관련

  - -i(--install)

  - v(verbose) : 설치 상세 내역

  - h(hash marks) : 설치시 progress를 \# 으로 표시

  - --nodeps : 의존성을 무시하고 설치 진행

  - --replacepkgs : 기존 설치된 동일 패키지가 있으면 패키지 교체

  - --replacefiles : 기존 설치된 동일 패키지가 있다면 파일 교체

  - --force : 강제로 설치 진행

- 업그레이드 관련

  - -U(--upgrade)

  - v

  - h

  - --nodeps, --replacepaks, --replacefiles, --force 지원

  - --oldpackage : 새로 설치할 패키지가 기존 설치된 패키지보다 낮은 버전이라도 설치

- 삭제 관련

  - -q(--query) : 특정 패키지 정보, 시스템에 설치된 패키지 정보 확인

  - a : 전체 패키지 정보 확인

  - i : 설치된 패키지 상세 정보 확인

  - ip : rpm 파일의 상세 정보 확인(설치전)

  - R : 설치된 패키지의 의존성 확인

  - Rp : rpm 파일의 의존성 확인

  - l : 설치된 패키지의 파일(들) 경로 확인

  - lp : rpm 파일의 설치파일(들) 경로 확인

  - f : 특정 파일을 설치한 패키지명 확인

- 패키지 정보 확인

# SSH Key 생성 및 설정

- ssh key 생성 및 배포

- ssh 공개키와 비공개키로 구성

- 공개키는 원격 서버에 비공개키는 ssh 클라이언트가 있는 머신에 위치

- 원격 빌드시에 jenkins가 있는 서버에는 비공개키를 빌드 대상 서버에는 공개키를 반영

1.  ssh key 생성

    - 리눅스 환경

      - ssh-keygen -t rsa (~/.ssh 에 id_rsa, id_rsa.pub 생성)

      - PEM option 으로 인증서 생성

        - ssh-keygen -t rsa -b 4096 -m PEM

2.  ssh 공개키 배포

    - id_rsa.pub 파일을 원격서버로 배포

    - scp \$HOME/.ssh/id_rsa.pub <a href="mailto:user@remote.server.com" class="external-link" rel="nofollow">user@remote.server.com</a>:id_rsa.pub -p(포트번호)

3.  id_rsa.pub 파일을 원격 서버로 복사

    - 원격서버의 접속할 계정의 home 디렉토리/.ssh/authorized_keys 에 복사

    - cat id_rsa.pub \>\> ~/.ssh/authorized_keys

    - chmod 700 ~/.ssh/authorized_keys

4.  원격서버 접속 설정

    - vi /etc/ssh/sshd_config

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      PermitRootLogin yes
      PasswordAuthentication yes
      ```

      </div>

      </div>

- passphase 물어 볼때 조치 방법

  - ssh-agent가 passphase를 저장하고 있지 않아 물어 보는 것

  - ssh-add ~/.ssh/id_rsa 수행해서 “Could not open a connection to you authentication agent “ 오류발생시 ssh-agent가 동작하지 않고 있는 것

  - eval \${ssh-agent} 명령으로 agent pid 출력 후
