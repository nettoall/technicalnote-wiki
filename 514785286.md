- GCP 환경에서 API 호출시 간헐적으로 발생 오류

  - 현상: 잦은 빈도가 아닌 간헐적으로 메일 송신시 처음 발생 → 대비책으로 Sevice 클래스에 @Retry 를 적용하여 한번 실패시 재전송하도록 조치됨

  - java.net.SocketException: Connection reset 오류 발생

  - org.springframework.web.client.ResourceAccessException: I/O error on POST request…

- 동일한 현상에 대해 stackoverflow 검색 결과 원인으로 다음을 추정함

  - proxy 설정

  - TLS1.2 관련(AWS에서) 설정 (확인 결과 간헐적으로 발생하는 오류 현상에는 적절한 조치 아님)

    - 권장 추가 로직(RestTemplate 사용시)

- 서베이 결과 다음과 같은 원인을 reset 원인으로 추정함

  1.  서버 부하로 인한 connection timeout

  2.  일시적인 네트워크 속도 문제

  3.  Client 에서 연결을 끊는 경우

  4.  서버에 설정된 maxHttpHeaderBody 옵션

  5.  메모리 이슈

- 참고로 서버쪽에서 Connection reset을 하게 되면 다음과 같은 오류가 발생됨

  - java.net.SocketException reset by peer

- <a href="https://stackoverflow.com/questions/62929/java-net-socketexception-connection-rest" class="external-link" rel="nofollow">https://stackoverflow.com/questions/62929/java-net-socketexception-connection-res</a>et 의 내용을 참고하면

  - 가장 간단한 방법은 소켓을 닫은 다음에 출력 스트림에 많은 데이터를 쓰는 것이다.

  - 소켓을 닫으면 상대방에서 대화가 끝났음을 알리고 상대방은 연결을 잊어 버릴 수 있습니다.

  - 어쨋든 해당 스트림에 더 많은 데이터를 보내면 피어는 RST로 이를 거부하여 듣고 있지 않음을 알립니다.

  - 다른 경우에는 개입하는 방화벽이나 원격 호스트 자체가 TCP 연결을 잃어 버릴 수도 있음

  - 이는 오랫동안 데이터를 보내지 않았거나 (일반적으로 2시간 초과) 피어가 재부팅되어 활성 연결에 대한 정보가 손실되었기에 발생할 수 있음. 이러한 존재하지 않는 연결 중 하나에서 데이터를 보내면 RST가 발생

- 결정적인 내용으로는 Cloud 에는 유휴 Connection 관리 설정이 있다고 함(참고: https://sillydev.tistory.com/74)

  - Azure Load Balancer 에서는 사용하지 않고 4분(기본값)이 지난 TCP 연결 닫음

  - AWS Gateway Load Balancer TCP 연결은 기본값이 350초, TCP가 아닌 연결에 대해서는 120초 유지

  - GCP WebSocket 연결 제한 시간은 부하 분산기 유형에 따라 다름. 기본 애플리케이션 부하 분산기의 경우 기본값은 30초 (https://cloud.google.com/load-balancing/docs/https?hl=ko)

- Connection 에서 일정 유휴 시간을 주고 그 시간이 지나면 client 에서 connection을 해제하도록 처리 (참고: <a href="https://backend-dev.tistory.com/189" class="external-link" rel="nofollow">https://backend-dev.tistory.com/189</a>) - 증상 해결됨

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Configuration
  public class WebConfig implements WebMvcConfigurer {

    @Bean
    public RestTemplate restTemplate() {
      RequestConfig requestConfig = RequestConfig.custom()
        .setCookieSpec(CookieSpecs.STANDARD)
        .build();

      HttpComponentsClientHttpRequestFactory factory = new HttpComponentsClientHttpRequestFactory();
      factory.setReadTimeout(20000);
      factory.setConnectTimeout(20000);

      HttpClient httpClient = HttpClientBuilder.create()
        .setMaxConnTotal(150)
        .setMaxConnPerRoute(70)
        .evictIdleConnections(60000, TimeUnit.MILLISECONDS)
        .evictExpiredConnections().setDefaultRequestConfig(requestConfig).build();
      factory.setHttpClient(httpClient);
      return new RestTemplate(factory);
    }
  }
  ```
