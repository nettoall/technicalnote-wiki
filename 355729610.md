# Docker Registry

- Registry : 컨테이너 이미지를 저장하는 저장소

- Docker hub : hub.docker.com

- Private Registry : 사내 컨테이너 저장소

## docker hub

- public docker registry

- url : https://hub.docker.com

- image 종류 : official image, verified publisher etc.

- docker search “keyword” 로 검색

## Private Registry

- docker hub에서 registry 다운 받아 private registry 구성\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ docker run -d -p 5000:5000 --restart always --name registry registry:2
  ```

  </div>

  </div>

- private registry에서 push/pull 을 하려면 repository 이름을 주어야 함

  - localhost:5000/ubuntu:18.04

  - <a href="http://docker.example.com:5000/ubuntu:18.04" class="external-link" rel="nofollow">docker.example.com:5000/ubuntu:18.04</a>

# 실습

## public Registry 사용

1.  이미지 검색 : docker search ‘keyword’

2.  이미지 다운 : docker pull ‘keyword’:'version'

3.  registry 로그인 : docker login

4.  public registry 에 등록하기 위해서는 id/keyword 형태로 이미지 이름 바꾸어야 함 : docker tag ‘keyword’:'version' id/'keyword':'version'

5.  registry 이미지 등록 : docker push id/‘keyword’:'version'

## Private Registry

1.  <a href="http://hub.docker.com" class="external-link" rel="nofollow">hub.docker.com</a> 에서 registry 다운 및 실행 : docker run -d -p 5000:5000 --restart always --name registry registry:2

2.  private registry에 저장한 이미지는 다음과 같은 이름을 가져야 함 : localhost:5000/httpd:lastest 등

    - 사내일 경우 localhost가 아닌 공식 명칭(<a href="http://hub.example.com" class="external-link" rel="nofollow">hub.example.com</a>) 가 같은 이름이어야 함

3.  tag 이름 만들기 : docker tag httpd:latest localhost:5000/httpd

4.  private registry에 push : docker push localhost:5000/httpd:latest

5.  조회 : /var/lib/docker/volumes/...../repositories 에 저장됨

# 컨테이너 사용

## 컨테이너 이미지 관리

### 이미지 검색

docker search \[옵션\] \[이미지이름:태그명\]

-f, --filter : filter

--format string

– limit int

--no-trunc

### 이미지 다운로드

docker pull \[옵션\] \[이미지이름:태그명\]

-a, --all-tags

--disable-content-trust

--platform string

-q, --quiet

### 저장된 이미지 조회

docker images

### 이미지 상세조회

docker inspect \[옵션\] \[이미지이름:태그명\]

-f, --format string

-s, --size

--type

### 이미지 삭제

docker rmi \[옵션\] \[이미지이름:태그명\]

-f, --force

--no-prune

## 컨테이너 실행 & 종료

### 컨테이너 생성

docker create \[옵션\] \[이미지이름:태그명\]

ex) docker create --name webserver nginx:1.14

### 컨테이너 실행

docker start \[옵션\] \[컨테이너이름\]

ex) docker start webserver

### 컨테이너 생성 & 실행(pull & 생성 & 실행)

docker run \[옵션\] \[이미지이름:태그명\]

ex) docker run --name webserver -d nginx:1.14

### 실행중 컨테이너 목록 확인

docker ps \[옵션\]

ex) docker ps

### 실행중 컨테이너 상세정보 조회

docker inspect \[컨테이너이름\]

ex) docker inspect webserver

### 컨테이너 중지

docker stop \[옵션\] \[컨테이너이름\]

ex) docker stop webserver

### 컨테이너 삭제

docker rm \[옵션\] 컨테이너이름

ex) docker rm webserver

### 실행중 컨테이너 프로세스 정보 조회

docker top \[옵션\] 컨테이너이름

docker top webserver

### foregroud로 실행중인 컨테이너 연결

docker attach \[옵션\] 컨테이너이름

ex) docker attach centos

### 동작중인 컨테이너에 new 명령어 추가 실행

docker exec \[옵션\] 컨테이너이름

ex) docker exec -it webserver /bin/bash

### 컨테이너 로그 조회

docker logs \[옵션\] 컨테이너이름

ex) docker logs webserver

docker logs -f

### 컨테이너내 프로세스 확인

docker top \[옵션\] 컨테이너이름

# 실습

## Docker Container 사용

1.  이미지 검색 : docker search nginx

2.  이미지 다운로드 : docker pull nginx:1.14

3.  다운받은 이미지 조회 : docker images

4.  이미지 full name 으로 표시 : docker images --no-trunc

5.  현재 모든 컨테이너 : docker ps -a

6.  docker container 생성 : docker create --name webserver nginx:1.14

7.  컨테이너 실행 : docker start webserver

8.  컨테이너 상세 정보 조회 : docker inspect webserver

9.  상세정보 중 필요한 정보만 조회 : docker inspect --format ‘{{.NetworkSettings.IPAddress}}’ webserver

10. alias 로 정보 조회를 만들 수 있음 : alias cip=”docker inspect…”

11. 현재 실행중 컨테이너 : docker ps

12. 로그 출력 : docker logs webserver

13. 로그 추적 : docker logs -f webserver

14. 컨테이너내 실행 프로세스 출력 : docker top webserver

15. 실행중 컨테이너 접속 : docker **exec** -it webserver /bin/bash

16. 컨테이너 내 디렉토리로 가서 수정하고자 하는 파일 수정

17. 컨테이너 종료 : docker stop webserver
