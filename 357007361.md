# 쿠버네티스 소개

## 쿠버네티스 구성 방법

1.  퍼블릭 클라우드 업체 제공 관리형 쿠버네티스

    1.  EKS(Amazon Elastic Kubernetes Service)

    2.  AKS(Azure Kubernetes Services)

    3.  GKE(Google Kubernetes Engine)

2.  OS 제공하는 설치형 쿠버네티스

    1.  Suse Rancher

    2.  Redhat OpenShift

3.  시스템에 쿠버네티스 클러스터를 자동으로 구성

    1.  Kubeadm : 제일 많이 알려짐

    2.  kops(Kubernetes Operations)

    3.  KRIB(Kubernetes Rebar Integrated Bootstrap)

    4.  kubespray

      #=============#

          cfg.vm.synced_folder "../data", "/vagrant", disabled: true 
          cfg.vm.provision "shell", path: "config.sh", args: N
          cfg.vm.provision "shell", path: "install_pkg.sh", args: [ Ver, "Main" ]
          cfg.vm.provision "shell", path: "master_node.sh"
        end

      #==============#
      # Worker Nodes #
      #==============#

      (1..N).each do |i|
        config.vm.define "w#{i}-k8s" do |cfg|
          cfg.vm.box = "sysnet4admin/CentOS-k8s"
          cfg.vm.provider "virtualbox" do |vb|
            vb.name = "w#{i}-k8s(github_SysNet4Admin)"
            vb.cpus = 1
            vb.memory = 2560
            vb.customize ["modifyvm", :id, "--groups", "/k8s-SgMST-1.13.1(github_SysNet4Admin)"]
          end
          cfg.vm.host_name = "w#{i}-k8s"
          cfg.vm.network "private_network", ip: "192.168.1.10#{i}"
          cfg.vm.network "forwarded_port", guest: 22, host: "6010#{i}", auto_correct: true, id: "ssh"
          cfg.vm.synced_folder "../data", "/vagrant", disabled: true
          cfg.vm.provision "shell", path: "config.sh", args: N
          cfg.vm.provision "shell", path: "install_pkg.sh", args: Ver
          cfg.vm.provision "shell", path: "work_nodes.sh"
        end
      end

    end

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    #!/usr/bin/env bash

    # vim configuration 
    echo 'alias vi=vim' >> /etc/profile

    # swapoff -a to disable swapping
    swapoff -a
    # sed to comment the swap partition in /etc/fstab
    sed -i.bak -r 's/(.+ swap .+)/#\1/' /etc/fstab

    # kubernetes repo
    gg_pkg="packages.cloud.google.com/yum/doc" # Due to shorten addr for key
    cat <<EOF > /etc/yum.repos.d/kubernetes.repo
    [kubernetes]
    name=Kubernetes
    baseurl=https://packages.cloud.google.com/yum/repos/kubernetes-el7-x86_64
    enabled=1
    gpgcheck=0
    repo_gpgcheck=0
    gpgkey=https://${gg_pkg}/yum-key.gpg https://${gg_pkg}/rpm-package-key.gpg
    EOF

    # Set SELinux in permissive mode (effectively disabling it)
    setenforce 0
    sed -i 's/^SELINUX=enforcing$/SELINUX=permissive/' /etc/selinux/config

    # RHEL/CentOS 7 have reported traffic issues being routed incorrectly due to iptables bypassed
    cat <<EOF >  /etc/sysctl.d/k8s.conf
    net.bridge.bridge-nf-call-ip6tables = 1
    net.bridge.bridge-nf-call-iptables = 1
    EOF
    modprobe br_netfilter

    # local small dns & vagrant cannot parse and delivery shell code.
    echo "192.168.1.10 m-k8s" >> /etc/hosts
    for (( i=1; i<=$1; i++  )); do echo "192.168.1.10$i w$i-k8s" >> /etc/hosts; done

    # config DNS  
    cat <<EOF > /etc/resolv.conf
    nameserver 1.1.1.1 #cloudflare DNS
    nameserver 8.8.8.8 #Google DNS
    EOF
    ```

### 마스터 노드

- kubectl

  - 쿠버네티스 클러스터에 명령내리는 역할.

  - API 서버와 주로 통신.

  - <u>마스터 노드 외에 다른 노드에도 설치 가능</u>

- API 서버

  - 쿠버네티스 클러스터 중심 역할을 하는 통로

  - 상태값을 저장하는 etcd와 주로 통신, 다른 요소들도 통신함.

  - 관리자 역할

- etcd

  - 구성 요소들의 상태값 저장.

  - 분산 저장이 가능한 key-value 저장소.

  - etcd 정보만 백업되어 있으면 장애 상황에서도 쿠버네티스 클러스터 복구 가능

  - 복제해 여러 곳에 저장해 두면 하나의 etc에서 장애가 나더라도 시스템 사용성 확보 가능

- 컨트롤러 매니저

  - 쿠버네티스 클러스터의 오브젝트 상태 관리.

  - 워커 노드와 통신이 되지 않을 경우, 상태 체크와 복구는 컨트롤러 매니저에 속한 노드 컨트롤러에서 담당.

- 스케줄러

  - 노드의 상태와 자원, 레이블, 요구 조건 등을 고려해 파드를 어떤 워커 노드에 생성할 것인지 결정, 할당.

  - 파드를 조건에 맞는 워커 노드에 지정.

  - 파드가 워코 노드에 할당되는 일정 관리

### 워커 노드

- kubelet

  - 파드의 구성 내용(PodSpec)을 받아서 컨테이너 런타임 전달

  - 파드 안의 컨테이너 모니터링

- 컨테이너 런타임(CRI, Container Runtime Interface)

  - 파드를 이루는 컨테이너 실행 담당

  - 파드안의 컨테이너 실행을 담당하는 표준 인터페이스

- 파드(POD)

  - 한 개 이상의 컨테이너로 단일 목적의 일을 하기 위해 모인 단위

  - 언제라도 죽을 수 있는 존재

### 구성요소

- 네트워크 플러그인

  - 쿠버네티스 클러스터의 통신을 위해 네트워크 플러그인 선택 및 구성 필요(필수)

  - CNI로 구성.

  - CNI 종류 : Calico, Flannel, cilium, Kube-router, Romana, WeaveNet, Canal

  - CNI

    - 클라우드 네이티브 컴퓨팅 재단 프로젝트

    - 컨테이너의 네트워크 안정성과 확장성을 보장하기 위해 개발

    - 목적에 맞게 네트워크 플러그인을 선택하여 구성

    - Calico : L3로 컨테이너 네트워크 구성

    - Flannel : L2로 구성 등 각각의 플러그인만의 특징이 있음

- CoreDNS

  - 클라우드 네이티브 컴퓨팅 재단에서 보증하는 프로젝트

  - <u>빠르고 유연한 DNS 서버</u>

  - 쿠버네티스 클러스터 구성시 도메인 네임을 편리하게 관리해 주는 CoreDNS 사용

- Kube-Proxy

  - 파드가 통신할 수 있는 네트워크 설정

## kubelet 장애시 pod 생성 관리 실습

### kubelet

- pod의 생성과 상태 관리 및 복구 담당

- kubelet이 문제가 생기면 pod가 정상적으로 관리 안 됨

1.  pod 생성 : m-k8s 에서 pod 생성

    - kubectl create -f ~/\_Book_k8sInfra/ch3/3.1.6/nginx-pod.yaml 실행

2.  pod 조회

    - kubectl get pod

    - kubectl get pods -o wide : 상세 정보 보기 IP, node 정보 나옴

3.  pod가 생성되어 실행중인 워커 노드 접속하여 kubelet 중지

    - systemctl stop kubelet

4.  m-k8s 에서 pod 상태 조회

    - kubectl get pod : 실행중인 상태로 표시됨

5.  m-k8s에서 pod 삭제 명령 실행

    - kubectl delete pod nginx-pod

    - 정상적으로 삭제가 안 됨 : 실행중인 상태에서 나와 pod 상태 조회하면 Terminating 상태임

6.  pod가 실행중인 워커 노드에 접속하여 kubelet 실행

    - systemctl start kubelet

7.  m-k8s 에서 pod 정보 조회

    - kubectl get pod

## kube-proxy 장애 시 통신 실습

### kube-proxy

- pod의 통신을 담당

- kube-proxy 장애 발생하여 현상 확인

1.  m-k8s 포드 생성하여 배포

    - kubectl create -f ~/\_Book_k8sInfra/ch3/3.1.6/nginx-pod.yaml

    - kubectl get pods -o wide : pod 상태 조회(ip 정보)

    - nginx 메인 페이지 내용 확인 : curl pod ip

2.  pod가 실행중인 워커 노드에 접속하여 br_netfilter 모듈 중지

    - modprobe -r br_netfilter

    - systemctl restart network

3.  m-k8s 에서 curl 명령으로 nginx 메인 페이지 다시 조회

    - curl pod ip : Failed connect to ….

    - pod 상태 조회 : kubectl get pods -o wide → 상태는 실행중으로 나옴

4.  pod 실행중인 워커 노드에서 br_netfilter 모듈 실행

    - modprobe br_netfilter

    - reboot

5.  m-k8s 에서 pod 정보 조회

    - kubectl get pods -o wide

    - curl pod ip

6.  pod 삭제

    - kubectl delete -f ~/\_Book_k8sInfra/ch3/3.1.6/nginx-pod.yaml
