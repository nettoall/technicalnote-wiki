# 에러 유형

## java.lang.linkageError 발생

### 현상

- Jeus log에 “SLF4J: Failed to load class ‘org.slf4j.impl.StaticLoggerBinder’ 가 찍힘

- 업무 log4j2.xml에 정의한 로그가 출력되지 않는 것으로 파악.

- log4j-slf4j-impl-2.1.jar 에 “org.slf4j.impl.StaticLoggerBinder” 가 정의되어 있는 것을 확인하여 was class loader 우선순위 문제로 의심하여 jeus-we-dd.xml 의 “webinf-first” 활성화시킴

- 서버 재시작시 “java.lang.LinkageError: loader constrint violation: when resolving overridden method “com.sun.xml.ws.wsdl.writer.WSDLGenerator\$JAXWSOutputSchemaResolver.createOutput(Ljava/lang/String;Ljava/lang/String;)Ljavax/xml/transform/Result;”….

### 원인파악

오류가 발생하는 클래스의 출처를 추적해 본 결과

- com.sun.xml.ws.wsdl.writer.WSDLGenerator jeus/lib/shared/wsit-2.3/webservices-rt-2.3.1.jar 에서 사용

- javax.xml.transform.Result : jeus/lib/endorsed/xml-apis.jar에서 사용

- javax.xml.transform.Result : app/lib/jaxp-api-1.4.2.jar에서 사용

- xml-apis.jar 와 jaxp-api-1.4.2.jar 는 둘 다 1.4 버전으로 동일한 javax.xml.transform.Result사용

- 추정 : javax.xml.transform.Result를 사용하는 또 다른 jar가 있는 것이 아닌가 ?

- 모든 javax.xml.transform.Result를 사용하는 라이브러리나 해당 패키지를 임시로 jar안에서 제거함으로써 테스트를 진행해 본 결과, 최종적으로 jdk/jre/lib/rt.jar 내의 javax.xml.transform.Result를 사용함을 확인

- 즉, 명확하게 java.lang.LinkageError의 원인 파악 실패

### 원인파악2

- SLF4J가 정상작동하지 않는 원인을 찾는 것으로 방향 전환

- TMAX Technet에서 “StaticLoggerBinder”로 검색해 보니 다음과 같은 결과가 나옴

- Jeus8.5에 새로운 기능을 추가하면서 slf4j 로그 기능 이슈가 발생하여 jeus_home\lib\system 의 두 개의 jar파일을 제거(백업)하고 서버 재기동하면 slf4j기능이 정상 동작한다는 답변 발견

- ehcache-scf-replication.jar-with-dependencies.jar, hazelcase-all-4.2.1.jar 파일을 백업(파일명에 '\_'을 붙여 비활성화) 한 후, 서버 재기동

- 로깅이 정상적으로 수행되는 것을 확인
