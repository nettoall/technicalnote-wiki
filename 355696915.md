# 컨테이너 리소스

- 기본적으로 컨테이너는 호스트 하드웨어 리소스의 사용 제한을 받지 않음

- 여러 docker를 실행하기 위해 실행시 제한을 둠

- 제한할 수 있는 리소스

  - CPU

  - Memory

  - Disk I/O

- docker run --help 로 리소스 옵션 참조

## 메모리 리소스 제한

- 제한 단위 b,k,m,g

<div class="table-wrap">

|  |  |
|----|----|
| **옵션** | **설명** |
| --memory, -m | 컨테이너가 사용할 최대 메모리 양을 지정 |
| --memory-swap | 컨테이너가 사용할 스왑 메모리 영역에 대한 설정 메모리 + 스왑. 생략시 메모리의 2배가 설정됨 |
| --memory-reservation | --memory값보다 적은 값으로 구성하는 소프트 제한 값 설정(지정한 메모리를 보장 최소) |
| --oom-kill-disable | OOM Killer가 프로세스 kill하지 못하도록 보호 |

</div>

ex)

docker run -d -m 512m nginx:1.14

docker run -d -m 1g --memory-reservation 500m nginx:1.14

docker run -d -m 200m --memory-swap 300m nginx:1.14

## 컨테이너 리소스 제한

### stress container 생성

- 컨테이너 빌드 : 부하 테스트 프로그램 stress를 설치하고 동작시키는 컨테이너 빌드

- cpu부하 테스트 : 2개의 cpu core를 100% 사용하도록 부하 발생 : stress --cpu 2

- 메모리 부하 테스트 : 프로세스 수 2개와 사용할 메모리만큼 부하 발생 : stress --vm 2 --vm-bytes \<사용할크기\>

<div class="code panel pdl" style="border-width: 1px;">

<div class="codeContent panelContent pdl">

``` syntaxhighlighter-pre
$ vi dockerfile
FROM debian
MAINTAINER STRESS
RUN apt-get update; apt-get install stress -y
CMD ["/bin/sh","-c","stress -c 2"]
```

</div>

</div>

- docker build

  - docker build -t stress .

### 메모리 리소스 제한

- swap 메모리 용량 제한이 실제 메모리 제한과 어떤 관련성이 있는지 확인\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ docker run -m 100m --memory-swap 100m stress:latest stress --vm 1 --vm-bytes 90m -t 5s
  $ docker run -m 100m --memory-swap 100m stress:latest stress --vm 1 --vm-bytes 150m -t 5s
  $ docker run -m 100m stress:latest --vm 1 --vm-bytes 150m -t 5s
  ```

  </div>

  </div>

- 1번 docker는 실행되나 2번 docker는 메모리가 150m 로 swap memory를 넘기 때문에 실행 안됨

- 3번 docker는 swap 메모리를 설정 안 했기 때문에 기본 메모리의 2배가 할당되어 실행됨

<!-- -->

- 컨테이너를 OOM-killer로부터 보호한다.

- OOM-killer disable 설정 확인\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ docker run -d -m 100M --name m4 --oom-kill-disable=true nginx
  $ cat /sys/fs/cgroup/docker/containerID/memory.com-control # oom 관련 disable 설정됨
  $ docker inspect m4 # memory, swap-memory, oom kill 확인 
  ```

  </div>

### CPU 리소스 제한

- cpu 개수를 제한하여 컨테이너를 실행(core 를 분산시켜 실행도 가능)\

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre

  $ htop
  $ docker rm c1
  ```

### Block I/O 제한

- 컨테이너에서 --device-write-iops를 적용해서 write 속도의 초당 quota를 제한해서 io write를 발생\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ docker run -it --rm --device-write-iops /dev/sda:10 ubuntu:latest /bin/bash 
  /# dd if=/dev/zero of=file1 bs=1M count=10 oflag=direct
  ```

  </div>

  </div>

- 사전에 lsblk 로 디바이스 명을 확인해 본다.

- 다음 write quota를 100으로 변경 후 같은 작업을 반복한다.\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ docker run -it --rm --device-write-iops /dev/sda:100 ubuntu:latest /bin/bash
  /# dd if=/dev/zero of=file1 bs=1M count=10 oflag=direct
  ```

  </div>

# Docker 볼륨

- 컨테이너 이미지는 read only

- 컨테이너에 추가되는 데이터는 별도의 RW 레이어에 저장\
  Docker run → rw 레이어가 붙어서 동작됨 Union File system 구조 ⇒ Overlay

- Docker 내 저장되는 데이터를 영구 보관하기 위해 Disk 에 저장하도록 설정\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ docker run -d -named b-v /dbdata:/var/lib/mysql -e MYSQL_ALLOW_EMPTY_PASSWORD = pass mysql: latest
  ```

  </div>

  </div>

  - /dbdata: 영구 저장소\

- 사용법

  - -v \<host path): \<container mount Path\>

  - -v \<host path\>: \<Container mount Path\> : \<read write mode\>

  - -v (Container mount path)\

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    $ docker run -d -v /dbdata:/var/lib/mysql - e MYSQL_PASSWORD=pass mysql:latest
    $ docker run -d -v /webdata:/webdata/www/html:ro httpd: latest #to read only mode
    $ docker run -d -v /var/lib/ mysql MYSQL.. PASSWORD = Pass Mysql:latest
    ```

    </div>

    </div>

  - 3 : 임의의 디렉토리를 만들어서 영구 저장

# 컨테이너 통신(네트워크)

## 컨테이너 네트워크 모델

- docker0 : docker daemon 실행시 network interface 생성

  - docker 간의 브리지 역할

  - virtual ethernet bridge: 172.17.0.0/16

  - L2 통신 기반

  - container 생성시 veth 인터페이스 생성(sandbox)

  - 모든 컨테이너는 외부 통신을 docker0 통해 진행

  - container running 시 172.17.X.Y 로 IP 할당

## 컨테이너 포트

### port-forwarding

- container port를 외부로 노출시켜 외부 허용 허용

- container 가 서비스하는 port 를 host 의 포트와 매핑시켜 서비스 가능하도록 함

- iptables rule을 통한 포트 노출

  - -p hostPort:cotainerPort

  - -p containerPort : hostPort를 random port 생성

  - -P : randomport\

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    $ docker run --name web -d -p 80:80 nginx:1.14
    $ iptables -t nat -L -n -v  # iptable 조회 명령어 
    ```

    </div>

    </div>

### 실습

<div class="code panel pdl" style="border-width: 1px;">

<div class="codeContent panelContent pdl">

``` syntaxhighlighter-pre
$ docker run -p 80:80 -d --name web1 nginx
$ docker ps # port 정보 확인 
$ docker run -p 80 -d --name web2 nginx  # 임의의 hostport 설정 
$ docekr ps # docker port 정보 확인
$ docker run -P -d --name web3:nginx # docker file에 지정된 expose port에 random host port 할당
$ docker ps # docker port 정보 확인 
```

## 컨테이너 네트워크 추가
