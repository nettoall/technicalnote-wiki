# 개념

- 오픈소스

- 고급 key-value 저장소

- 고성능의 확장 가능한 웹 어플리케이션 구축에 적합한 솔루션

## 세가지 주요 특성

- 지속성을 위해서만 디스크를 이용하여 데이터베이스 전체를 메모리에 보관

- 많은 key-value 데이터 저장소와 비교할 때 상대적으로 풍부한 데이터 유형 세트를 가지고 있음

- 원하는 수의 슬레이브에 데이터를 복제할 수 있음

## 장점

- 매우 빠름 : 초당 약 11000SET, 약 81000 GET 을 수행할 수 있음

- 풍부한 데이터 유형 지원 : 목록, 세트, 정렬 세트, 헤시 등 개발자가 이미 알고 있는 대부분의 데이터 유형 기본 지원

  - 어떤 데이터 유형이 문제를 잘 처리할 수 있는지 알수 있으므로 다양한 문제 쉽게 해결

- 작업은 원자적 : 모든 redis 작업은 원자적이므로 두 클라이언트가 동시에 엑세스 하는 경우, redis 서버가 업데이트된 값을 받게 됨

- 다중 유틸리티 도구 : redis 는 웹 어플리케이션 세션, 웹 페이지 조회 수 같이 단기 데이터나 캐싱, 메시지 큐와 같인 다양하게 사용할 수 있음

## 기타 Key-Value 저장소 비교

- Redis는 Value에 원자적 명령이 정의된 더 복잡한 데이터 유형을 포함할 수 있는 색다르게 진화된 key-value db 이다. (Value에 복잡한 데이터 유형을 가짐)

- redis는 메모리 내 데이터 베이스 이지만 디스크 데이터 베이스에 지속적으로 존재하므로 메모리보다 클 수 없는 데이터 세트의 제한으로 매우 높은 쓰기와 읽기 속도 다성

- 인메모리 데이터베이스의 또 다른 장점은 복잡한 데이터 구조의 메모리 표현이 디스크의 동일한 데이터 구조에 비해 조작하기가 훨씬 간단.

  - Redis는 내부 복잡성이 거의 없어 많은 작업 수행 가능

### redis 설정

1.  redis 설치 디렉토리의 redis.conf 편집

2.  CONFIG 명령으로 설정

  - redis 127.0.0.1:6379\> CONFIG GET \*

- 설정 편집하기

  - redis 127.0.0.1:6379\> CONFIG SET CONFIG_SETTING_NAME NEW_CONFIG_VALUE

  - 예제

    <div class="code panel pdl" style="border-width: 1px;">

    ``` syntaxhighlighter-pre
    OK

    1) "loglevel"
    2) "notice"
    ```

## redis의 데이터 유형

### String

- bytes 시퀀스 :

  - redis에서의 String은 binary safe : 특정 종료 문자로 결정되지 않는 알려진 길이를 가지고 있기 때문에 따라서 하나의 String에 512메가바이트까지 저장 가능

  - 예

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    redis 127.0.0.1:6379> SET name "tutorialspoint"
    OK
    redis 127.0.0.1:6379> GET name
    "tutorialspoint"
    ```

    </div>

    </div>

### Hashes

- key value 컬렉션

- redis hashes 는 문자열 필드와 문자열 값 간의 맵

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  redis 127.0.0.1:6379> HMSET user:1 username tutorialspoint password
  turotialspoint points 200
  OK
  redis 127.0.0.1:6379> HGETALL user:1
  1) "username"
  2) "tutorialspoint"
  3) "password"
  4) "tutorialspoint"
  5) "points"
  6) "200"
  ```

  </div>

  </div>

- HMSET, HGETALL redis 명령어

- 모든 hash는 2의 32승 -1 field-value pair로 저장 가능 (4 billion 이상)

### Lists

- Strings 목록

- 삽입한 순서대로 정렬됨

- list의 head나 tail로 입력 가능

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  redis 127.0.0.1:6379> lpush tutoriallist redis
  (integer) 1
  redis 127.0.0.1:6379> lpush tutoriallist mongodb
  (integer) 2
  redis 127.0.0.1:6379> lpush tutoriallist rabbitmq
  (integer) 3
  redis 127.0.0.1:6379> lpush tutoriallist 0 10
  1) "rabbitmq"
  2) "mongodb"
  3) "redis"
  ```

  </div>

  </div>

- list의 최대 길이는 2의 32승 -1 요소(4294967295, 4 billion 이상 list 마다)

### Sets

- string의 정렬되지 않은 컬렉션

- sets에서 요소를 추가, 삭제, 테스트시 O(1) 시간 복잡도

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  redis 127.0.0.1:6379> sadd tutoriallist redis
  (integer) 1
  redis 127.0.0.1:6379> sadd tutoriallist mongodb
  (integer) 1
  redis 127.0.0.1:6379> sadd tutoriallist rabbitmq
  (integer) 1
  redis 127.0.0.1:6379> sadd tutoriallist rabbitmq
  (integer) 0
  redis 127.0.0.1:6379> smembers tutoriallist
  1) "rabbitmq"
  2) "mongodb"
  3) "redis"
  ```

  </div>

  </div>

- 위의 예제에서 rabbitMq가 두번 추가되었으나 중보 허용하지 않음

- set의 최대 맴버는 2의 32승 -1 요소(4294967295, 4 billion 이상 set 마다)

### Sorted Sets

- Redis Sets와 유사하나 반복되지 않는 문자열 컬렉션

- Sorted Set의 모든 맴버들이 작은 수부터 큰 수까지 sorted sort가 정렬리 되도록 하는 ssocre와 연결됨

- 맴버는 고유하나 score는 반복될 수 있음

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  redis 127.0.0.1:6379> sadd tutoriallist redis
  (integer) 1
  redis 127.0.0.1:6379> sadd tutoriallist mongodb
  (integer) 1
  redis 127.0.0.1:6379> sadd tutoriallist rabbitmq
  (integer) 1
  redis 127.0.0.1:6379> sadd tutoriallist rabbitmq
  (integer) 0
  redis 127.0.0.1:6379> ZRANGEBYSCORE tutoriallist 0 1000
  1) "redis"
  2) "mongodb"
  3) "rabbitmq"
  ```

  </div>

  </div>

- 위의 예제에서 rabbitMq가 두번 추가되었으나 중보 허용하지 않음

- set의 최대 맴버는 2의 32승 -1 요소(4294967295, 4 billion 이상 set 마다)

## Redis - keys

- Redis key를 관리하는데 사용하는 명령어

- 사용법

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  redis 127.0.0.1:6379> SET tutorialspoint redis
  OK
  redis 127.0.0.1:6379> DEL tutorialspoint
  (integer) 1
  ```

  </div>

  </div>

- tutorialspoint 가 key 이면 삭제하는 명령어 DEL

- 존재하는 key를 삭제하면 (integer) 1, 없으면 (integer) 0 리턴

### 명령어

- DEL key : key 삭제

- DUMP key : 키에 저장된 직렬화된 값 리턴

- EXISTS key

- EXPIRE key seconds

- EXPIREAT key timestamp

- PEXPIRE key milliseconds

- PEXPIREAT key milliseconds-timestamp

- KEYS pattern

- MOVE key db

- PERSIST key

- PTTL key

- TTL key

- RANDOMKEY

- RENAME key newkey

- RENAMENX key newkey

- TYPE key
