# Docker Compose

- 여러 컨테이너를 일괄적으로 정의하고 실행할 수 있는 툴

- 하나의 서비스를 운영하기 위해 여러 어플리케이션을 실행

- 컨테이너화 된 어플리케이션을 통합 관리

- yaml 파일로 정의

- docker compose version 별로 사용가능한 정보에 따라 선택

## Docker compose 설치

- docker engine이 설치된 경우, docker repository를 사용하여 docker-compose plugin 만 설치\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ sudo apt-get update
  $ sudo apt-get install docker-compose-plugin
  ```

  </div>

  </div>

- 특정 버전의 docker-compose plugins 설치시\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ apt-cache madison docker-compose-plugin # 설치 가능한 버전의 docker-compose 조회 
  $ sudo apt-get install docker-compose-plugin=<VERSION_STRING>
  $ docker compose version # docker compose 버전 보기 
  ```

  </div>

  </div>

- manual 방식으로 docker-compose plugins 설치(linux 설치 권장방법)\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ DOCKER_CONFIG=${DOCKER_CONFIG:-$HOME/.docker}
  $ mkdir -p $DOCKER_CONFIG/cli-plugins
  $ curl -SL https://github.com/docker/compose/releases/download/v2.7.0/docker-compose-linux-x86_64 -o $DOCKER_CONFIG/cli-plugins/docker-compose
  $ chmod +x $DOCKER_CONFIG/cli-plugins/docker-compose
  ```

  </div>

  </div>

- 모든 사용자가 사용 하도록 docker-compose 설치\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ sudo mkdir -p /usr/local/lib/docker/cli-plugins
  $ sudo curl -SL https://github.com/docker/compose/releases/download/v2.7.0/docker-compose-linux-x86_64 -o /usr/local/lib/docker/cli-plugins/docker-compose
  $ sudo chmod +x /usr/local/lib/docker/cli-plugins/docker-compose
  ```

  </div>

  </div>

  이후 /etc/profile 에 /usr/local/lib/docker/cli-plugins 추가 필요

## docker compose 실행

- 웹서버를 동작하기 위한 예제

  1.  서비스 디렉토리 생성\

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      $ mkdir webserver
      $ cd webserver
      ```

      </div>

      </div>

  2.  docker-compose yaml 생성\

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      $ cat > docker-compose.yml
      version: "3"
      services;
        web:
          image; httpd:latest
          ports:
            - "80:80"
          links:
            - mysql:db
          command: apachectl -DFOREGROUND
          mysql:
            image: mysql:latest
            command: mysqld
            environment:
              MYSQL_ROOT_PASSWORD: pass
      ```

      </div>

      </div>

  3.  docker-compose 실행\

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      $ docker-compose up -d
      $ docker-copose ps
      $ docker-compose scale mysql=2   # mysql 개수 2개로 설정 
      $ docker-compose ps
      $ docker-copose down
      ```

      </div>

      </div>

## docker-compose 명령어

- docker-compose up : 컨테이너 생성/시작

  - docker-compose up -d : detach 모드 (백그라운드)

  - docker-compose -f /direcotry/docer-compose.yml

- docker-compose ps : 컨테이너 목록 조회

- docker-compose logs : 컨테이너 로그 출력

  - docker-compose logs 서비스이름

- docker-compose run : 컨테이너 실행

  - docker-compose run 서비스이름 실행명령어

- docker-compose start : 컨테이너 시작

- docker-compose stop : 컨테이너 중지

- docker-compose restart : 컨테이너 재시작

- docker-compose pause: 컨테이너 일시 정지

- docker-compose unpause: 컨테이너 재개

- docker-compose port : 공개 포트 번호 표시

- docker-compose config : 구성확인

- docker-compose kill : 실행중인 컨테이너 강제 정지

- docker-compose rm : 컨테이너 삭제

- docker-compose down : 리소스 삭제

- docker-compose sclae 서비스이름=개수 : 서비스 확장

2.  Dockerfile 생성\

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    # syntax=docker/dockerfile:1
    FROM python:3
    ENV PYTHONDONTWRITEBYTECODE=1
    ENV PYTHONUNBUFFERED=1
    WORKDIR /code
    COPY requirements.txt /code/
    RUN pip install -r requirements.txt
    COPY . /code/
    ```

    </div>

    </div>

3.  requirement.txt 생성 하여 저장\

4.  yml 파일 생성 (docker-compose.yml)\

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    version: "3.9"
       
    services:
      db:
        image: postgres
        volumes:
          - ./data/db:/var/lib/postgresql/data
        environment:
          - POSTGRES_DB=postgres
          - POSTGRES_USER=postgres
          - POSTGRES_PASSWORD=postgres
      web:
        build: .
        command: python manage.py runserver 0.0.0.0:8000
        volumes:
          - .:/code
        ports:
          - "8000:8000"
        environment:
          - POSTGRES_NAME=postgres
          - POSTGRES_USER=postgres
          - POSTGRES_PASSWORD=postgres
        depends_on:
          - db
    ```

    </div>

    </div>
