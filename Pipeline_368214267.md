# 정의

- build process를 정의하고 빌드하기 위한 stage를 제공, 테스트 하고 기능 제공

# 사용방법

- jenkinsfile 을 만들고 사용

- 브랜치 작업가 PR

- docker와 pipeline

## 참고

- https://velog.io/@bbkyoo/Jenkins-Pipeline-%EB%AC%B8%EB%B2%95-%EC%A0%95%EB%A6%AC-u825sfpf

- 관련 링크 : https://www.jenkins.io/doc/book/managing/plugins/

- Pipeline plugins: workflow-aggregator

## 구현 방법

- Jenkins Webadmin: Jenkins Web 에서 pipeline을 구성하여 빌드

- Git SCM : Git Repository에 Jenkinsfile 작성하여 빌드

- Blue Ocean: Pipeline을 시각화하여 손쉽게 구성 빌드

## Script 문법

- pipeline 정의는 최상위 pipeline

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  pipeline {

  }
  ```

  </div>

  </div>

- pipeline은 seciont, directives, steps 또는 assignment로 구성

### Section

#### agent

- agent : agent를 추가할 경우에만 설정

  - agent는 any, none, label, node, docker, dockerfile, kubernetes를 파라미터로 포함

  - agent는 pipeline 최상위에 포함되어야 함. none 으로 작성되었을 경우에는 stage에 포함되어야 함

  - docker 예제

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    agent {
      docker {
        image 'myregistry.com/node'
        label 'my-defined-label'
        registryUrl 'https://myregistry.com/'
        registryCredentialsId 'myPredefinedCredentialsInJEnkins'
      }
    }
    ```

    </div>

    </div>

- dockerfile 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  agent {
    dockerfile {
      filename 'Dockerfile.build'
      dir 'build'
      label 'my-defined-label'
      registryUrl 'https://myregistry.com/'
      registryCredentialsId 'myPredefinedCredentialsInJenkins' 
    }
  }
  ```

  </div>

  </div>

- agent가 적용된 JenkinsFile 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  pipeline {
    agent {
      stages {
        stage ('Example Build') {
          agent { docker 'maven:3-alpine' }
          steps {
            echo 'Hello, Maven'
            sh 'mvn --version'
          }
        }
        stage ( 'Example Test') {
          agent { docker 'openjdk:8.jre' }
          steps {
            echo 'Hello, JDK'
            sh 'java -version'
          }
        }
      }
    }
  }
  ```

  </div>

  </div>

- agent none이 pipeline 최상위 정의 시 stage 는 각각 agent를 포함하여야 함.

#### post

- 특정 stage 나 pipeline이 시작되기 이전 또는 이후에 실행될 condition block을 정의

- always, changed, fixed, regression, aborter, failure, success, unstable, unsuccessful, cleanup 등의 상태 정의

- 일반적으로 post는 pipeline 끝에 정의

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  pipeline {
    agent any
    stages {
      stage('Example') {
        steps {
          echo 'Hello, World'
        }
      }
    }
    post {
      always {
        echo 'I will always say Hello again'
      }
    }
  }
  ```

  </div>

  </div>

#### stages

- 하나 이상의 stage 에 대한 모음 정의

- pipeline 블록 안에서 한번만 실행됨

- 여러 stage 정의 가능

#### steps

- stage 내부에서 실행되는 단계

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  pipeline {
    agent any
    stages {
      stage('Example') {
        steps {
          echo 'Hello World'
        }
      }
    }
  }
  ```

  </div>

### directives

#### environment

- key-value 형태로 환경 변수 정의

- pipeline 안과 stage 안에 정의하는 것에 따라 적용 범위가 달라짐

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  pipeline {
    agent any
    environment {
      CC = 'clang'
    }
    stages {
      stage('Example') {
        environment {
          AN_ACCESS_KEY = credentials('my-prefined-secret-text')
        }
        steps {
          sh 'printenv'
        }
      }
    }
  }
  ```

  </div>

  </div>

#### options

- pipeline의 옵션

- pipeline 에서 한번만 정의

  1.  buildDiscarder

      - 특정 수의 최근 파이프 라인 실행에 대한 아티팩트 및 콘솔 출력 유지

      - 예제

        <div class="code panel pdl" style="border-width: 1px;">

        <div class="codeContent panelContent pdl">

        ``` syntaxhighlighter-pre
        options { buildDiscarder(logRotator(numToKeepStr: '1'))}
        ```

        </div>

        </div>

  2.  checkoutToSubdirectory

      - 작업 공간의 서브 디렉토리에서 자동 소스 체크아웃 실행

      - 예제

        <div class="code panel pdl" style="border-width: 1px;">

        <div class="codeContent panelContent pdl">

        ``` syntaxhighlighter-pre
        options { checkoutToSubdirectory('foo')}
        ```

        </div>

        </div>

  3.  disableConcurrentBuilds

      - 파이프 라인의 동시 실행을 허용하지 않음

      - 공유 리소스 등에 대한 동시 엑세스를 방지할 수 있음

      - 예제

        <div class="code panel pdl" style="border-width: 1px;">

        <div class="codeContent panelContent pdl">

        ``` syntaxhighlighter-pre
        options { disableConcurrentBuilds()}
        ```

        </div>

        </div>

  4.  disableResume

      - 마스터(?)가 다시 시작되면 파이프 라인을 다시 시작하지 않게 함

      - 예제

        <div class="code panel pdl" style="border-width: 1px;">

        <div class="codeContent panelContent pdl">

        ``` syntaxhighlighter-pre
        options { disableResume()}
        ```

        </div>

        </div>

  5.  newContainerPerStage

      - docker 또는 dockerFile 지정된 경우, Stage 마다 새 컨테이너 인스턴스에됨

  6.  overrideIndextriggers

      - 분기 인덱싱 트리거의 기본 처리를 재정의

      - 멀티 브랜치 또는 조직 레이블에서 분기 인덱싱 트리러가 비활성화된 경우에 대해서만 활성화됨

      - 예제

        <div class="code panel pdl" style="border-width: 1px;">

        <div class="codeContent panelContent pdl">

        ``` syntaxhighlighter-pre
        options { overrideIndexTriggers(true)}
        ```

        </div>

        </div>

  7.  preserveStashes : stage를 다시 시작할 때 사용할 수 있도록 완료된 빌드에서 숨김 유지

  8.  quietPeriod: 초단위 설정

  9.  retry: 실패시 재시도

  10. skipDefaultCheckout: 소스 체크아웃 건너뛰기

  11. skipStagesAfterUnstable: 빌드 불안정시 단계 스킵

  12. timeout: 파이프라인 실행에 대한 제한 시간 설정

  13. timestamps: 콘솔 출력에 수행된 시간 추가

  14. parallelsAlwaysFailFase: 모든 후속 병렬 단계에 대해 failfast true 설정

#### parameters

- 사용자 정의 변수

- string, text, boolean, choice, password 등을 정의

- parameters는 pipeline 에서 한번만 정의

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  pipeline {
    agent any
    parameters {
      string(name: 'PERSON', defaultValue: 'Mr Jenkins', description: 'Who should i Say Hello to?')
      text(name: 'BIOGRAPHY', defaultValue: '', description: 'Enter some informations about the person')
      booelanParam(name: 'TOGGLE', defaultValue: true, description: 'Toggle this value')
      choice(name: 'CHOICE', choices: ['One', 'Two', 'Three'], description: 'Pick something')
      password(name: 'PASSWORD', defaultValue: 'SECRET', description: 'Who should i Say Hello to?')
    }
    stages {
      stage('Example') {
        steps {
          echo "Hello ${params.PERSON}"
          echo "Biography: ${params.BIOGRAPHY}"
          echo "Toggle: ${params.TOGGLE}"
          echo "Choice: ${params.CHOICE}"
          echo "Password: ${params.PASSWORD}"
        }
      }
    }
  }
  ```

  </div>

  </div>

#### triggers

- 실행하는 방법에 대해 정의

- cron, pollSCM, upstream 등 여러 방식 정의 가능

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  pipeline {
    agent any
    triggers {
      cron('H */4 * * 1-5')
    }
    stages {
      stage('Example') {
        steps {
          echo 'Hello World'
        }
      }
    }
  }
  ```

  </div>

  </div>

- 지원 툴 정의

- maven, jdk, gradle 등

- tool 에 정의된 이름은 jenkins 관리의 Global Tool Configuration에 정의가 되어야 함

- 예제

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
    agent any
  ```

#### input

- 입력이 필요할 때 정의

- message, id, ok, submitter, submitterParameter, parameters 정의

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  pipeline {
    agent any
    tools {
      maven 'apache-maven-3.0.1'
    }
    stages {
      stage('Example') {
        input {
          message "Should we continue?"
          ok "Yes, we should"
          sumitter 'alice, bob'

            string(name: 'PERSON', defaultValue:'Mr Jenkins', description:'Who sould')
          }
        }
        steps {
          echo "Hello, ${PERSON}, nice to meet you"
        }
      }
    }
  }
  ```

## 실행 방법

- 순차 실행과 병렬 실행

- 기본적으로 순차 실행.

- paraller {} 정의 해서 병렬 수행 가능

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  pipeline {
    agent any
    stages {
      stage('Non-Sequential Stage') {
        agent {
          label 'for-non-sequential'
        }
        steps {
          echo "Hello, ${PERSON}, nice to meet you"
        }
      }
      stage('Sequential') {
        agent {
          label 'for-sequential'
        }
        encironment {
          FOR_SEQUENTIAL = "some-value"
        }
        stages {
          stage('In Sequential 1') {
            steps {
              echo "In Sequential 1"
            }
          }
          stage('In Sequential 2') {
            steps {
              echo "In Sequential 2"
            }
          }
          stage('Parallel In Sequential') {
            parallel {
              stage('In Paraller 1') {
                steps {
                  echo "In Paraller 1"
                }
              }
              stage('In Paraller 2') {
                steps {
                  echo "In Paraller 2"
                }
              }   
            }
          }        
        }
      }
    }
  }
  ```

  </div>

  </div>

- 예제2

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  pipeline {
    agent any
    stages {
      stage('Non-Parallel Stage') {
        steps {
          echo 'This stage will be executed first.'
        }
      }
      stage('Parallel Stage') {
        when {
          brance 'master'
        }
        failFase true
        paraller {
          stage('Branch A') {
            agent {
              label "for-branch-a"
            }
            steps {
              echo "On Branch A"
            }
          }
          stage('Branch B') {
            agent {
              label "for-branch-b"
            }
            steps {
              echo "On Branch B"
            }
          }
          stage('Branch C') {
            agent {
              label "for-branch-c"
            }
            stages {
              stage('Nested 1') {
                steps {
                  echo "Instage Nested 1 within Brance C"
                }
              }
              stage('Nested 2') {
                steps {
                  echo "Instage Nested 2 within Brance C"
                }
              }
            }
          }
        }
      }
    }
  }
  ```
