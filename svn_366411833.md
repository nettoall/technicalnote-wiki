## 권장버전

개발서버 구성시 svn server는 svn 1.9.x를 설치할 것을 강력히 권장함

같이 첨부한 svn server는 개발서버용으로 첨부함

## 버전 호환

copying file format 이 1.8부터 1.14까지 호환이 되므로, VisualSVN Server 4.3.1 을 설치 후, Jenkins 빌드까지 실행해 본 결과 문제가 없어 현재 4.3.1로 설치하였고 Subversion의 version 은 1.14.0

# SVN Repository

## 생성하기

svnadmin create ./mofa_temp_repo

각 repo하위의 conf/ 디렉토리의 authz, passwd, svnserve.conf 파일 편집

# svn hook

디렉토리 : */var/www/svn/myrepos/hooks/*

\*.tmpl 에서 tmpl 확장자 제거

hook 환경 설정 : hooks-env 에 설정

계정이 .profile bash_profile 설정 내역 적용 안 됨

# svn dump

## 백업받기

### 전체

svnadmin dump /home/svn/repo \> repo.dump

### 지정 리비전만 덤프 파일 생성

svnadmin dump /home/svn/repo -r 10 \> remp-10.dump

### 리비전 구간별 덤프 파일 생성

svnadmin dump /home/svn/repo -r 0:20 \> repo-0-20.dump

### 특정 날짜부터 현재까지 덤프 파일 생성

svnadmin dump /home/svn/repo -r {2022-05-15}:HEAD \> repo-20220515-now.dump

## 복원하기

svnadmin load /home/svn/repo \< repo.dump

### 리비전 복원시에는 순서 중요

svnadmin load /home/svn/repo \< repo-0-20.dump

svnadmin load /home/svn/repo \< repo-21-40.dump

복원할때는 레포지토리를 갓 생성한 상태에서 복원하는 것을 추천
