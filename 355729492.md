# Docker 구성 요소

- Docker Host(Linux Kernel)

- Docker Daemon : docker engine

  - systemctl status docker : 실행여부 확인

- Docker Client Command

- Docker Hub

- Container Images

- Container

## Container

### 개념

- Container

  - 하나의 Application 프로세스

  - cpu, memory, network, disk 각각 구성됨

  - base image layer, source image layer, app layer 로 구성됨

  - 동작중인 상태

  - docker daemon 위에서 동작

## Container image

- Container가 file로 저장된 상태

## 컨테이너 동작방식

- <a href="http://hub.docker.com" class="external-link" rel="nofollow">hub.docker.com</a> : 컨테이너 저장된 hub 사이트 (정확히는 container image가 아닌가)

- hub에서 docker를 검색하고 실행 절차

  - hub에서 nginx 검색 : docker search nginx

  - image 가져오기 : docker pull nginx:latest

  - docker 실행 : docker run -d --name web -p 80:80 nginx:latest

## Docker 실습

1.  \[root 계정\] cd /var/lib/docker

    1.  cd overlay2 : docker container image layer 저장 디렉토리

    2.  docker images : 저장된 이미지 조회

2.  \[docker 계정\] nginx 검색 : docker search nginx

3.  \[docker 계정\] nginx pull : docker pull nginx

4.  \[docker 계정\] image 조회 : docker image ls

5.  \[docker 계정\] docker 실행 : docker run --name web -d -p 80:80 nginx

6.  \[docker 계정\] 현재 동작중 docker 프로세스 확인 : docker ps

7.  \[docker 계정\] 테스트 : curl localhost:80

8.  \[docker 계정\] docker 중지 : docker stop web

9.  \[docker 계정\] docker 삭제 : docker rm web

10. \[docker 계정\] docker image 삭제 : docker rmi nginx

# Docker 만들기

- 개발 프로그램과 실행환경을 container 로 구성

- MSA(Micro Service Architectur) 환경의 polyglot 애플리케이션 운영

- (기능별, 모듈별?) Application Service를 Container 로 구성

## Dockerfile

- 쉽고, 간단, 명확한 구문을 가진 text file로 Top-Down 해석

- 컨테이너 이미지를 생성할 수 있는 고유의 지시어(Instruction)를 가짐

- 대소문자 구분하지 않으나 가독성을 위해 사용\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  $ mkdir build
  $ cd build
  $ vi dockerfile
  FROM node:12
  COPY hello.js /
  CMD ["node', '/hello.js'"]
  $ docker build -t imagename:tag .
  ```

  </div>

  </div>

- 지시어

<div class="table-wrap">

|            |                                                               |
|------------|---------------------------------------------------------------|
| **지시어** | **설명**                                                      |
| \#         | 주석                                                          |
| FROM       | 컨테이너의 BASE IMAGE                                         |
| MAINTAINER | 이미지를 생성한 사람의 이름 미                                |
| LABEL      | 컨테이너 이미지에 컨테이너의 정보 저장                        |
| RUN        | 컨테이너 빌드를 위해 base image에서 실행할 commands           |
| COPY       | 컨테이너 빌드시 호스트의 파일을 컨테이너로 복사               |
| ADD        | 컨테이너 빌드시 호스트의 파일(tar,url 포함)을 컨테이너로 복사 |
| WORKDIR    | 컨테이너 빌드시 명령이 실행될 작업 디렉토리 설정              |
| ENV        | 환경변수 지정                                                 |
| USER       | 명령 및 컨테이너 실행시 적용할 유저 설정                      |
| VOLUME     | 파일 도는 디렉토리를 컨테이너의 디렉토리로 마운트             |
| EXPOSE     | 컨테이너 동작 시 외부에서 사용할 포트 지정                    |
| CMD        | 컨테이너 동작 시 자동으로 실행할 서비스나 스크립트 지정       |
| ENTRYPOINT | CMD와 함께 사용하면서 command 지정시 사용                     |

</div>

## 우분투 웹서버 docker 만들기

1.  디렉토리 생성

    - mkdir webserver

2.  dockerfile 생성

    - webserver 디렉토리에서 vim Dockerfile 생성

    - dockerfile 내 명령어 한 줄 당 이미지가 하나씩 만들어짐 =\> 한 라인에 여러 명령어 나열하는게 적은 이미지를 생성함\

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      FROM ubuntu:18:04
      LABEL maintainer="Blue Cat <bludcat@gmail.com>"
      # install apache webserver
      RUN apt-get update 
      RUN apt-get install -y apache2
      RUN echo "TEST WEB" > /var/www/html/index.html
      EXPOSE 80
      CMD ["/usr/sbin/apache2ctl", "-DFOREGROUD"]
      ```

      </div>

      </div>

    - 라인 수 줄인 예

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      FROM ubuntu:18:04
      LABEL maintainer="Blue Cat <bludcat@gmail.com>"
      # install apache webserver
      RUN apt-get update \
          && apt-get install -y apache2
      RUN echo "TEST WEB" > /var/www/html/index.html
      EXPOSE 80
      CMD ["/usr/sbin/apache2ctl", "-DFOREGROUND"]
      ```

      </div>

      </div>

3.  dckerfile build\

## 스크립트 수행하는 docker 만들기

1.  작업 디렉토리 생성\

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    $ mkdir webpage
    ```

    </div>

    </div>

2.  webpage.sh 스크립트 생성\

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    #!/bin/bash
    mkdir /htdocs
    while:
    do
      /usr/games/fortune > /htdocs/index.html
      sleep 10
    done
    ```

    </div>

    </div>

3.  Dockerfile 생성

    1.  base image : debian

    2.  컨테이너에 fortune 어플리케이션 설치

    3.  webpage.sh 스크립트 수행\

        <div class="code panel pdl" style="border-width: 1px;">

        <div class="codeContent panelContent pdl">

        ``` syntaxhighlighter-pre
        FROM debian:latest
        RUN apt-get update \
          && apt-get install -y fortune
        COPY webpage.sh /
        CMD["bash","/webpage.sh"]
        ```

        </div>

        </div>

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">
