# DevOps Tools

<a href="https://github.com/showcases/devops-tools" class="external-link" data-card-appearance="inline" rel="nofollow">https://github.com/showcases/devops-tools</a>

- doker

- ansible

  - 자동화 어플리케이션 배포툴

  - playbook yml 파일로 config 설정

- kubernates

  - docker container management tool

- netdata

  - server monitoring tool

- vagrant

  - vm image manage tool

# Build 자동화

- Travis CI

  - https://travis-ci.org

  - github plugin

  - 설치 or 서버 필요 없이 github 계정으로 사용하여 세팅 완료

  - 소스를 서버에 pull 하면 자동으로 테스팅/빌드/deploy

  - project 폴더에 .travis.yml 파일 생성

  - gitlab 도 gitlab ci 라는 플러그인 형식 제공

- AWS DevOps

  - AWS CodeDeploy 같은 서비스로 배포 자동화 지원

  - AWS CodePipeline 으로 배포 단계별 진행 가능

  - On Premise 환경 지원(부분 유료)

# Unit Test

- Canary analysis

  - 넷플릭스 사용 툴

  - 운영과 비슷한 환경 구축 후 bot으로 서버 상태 측정

- Selenium

  - 단순한 Mock 객체로 실제 브라우저로 unit 테스트 지원

  - WebDriver로 여러 브라우저에서 테스트 지원

  - Unit Test 구현 또는 브라우저 plugin 형태로도 지원

  - 단점 : 서버 구축 필요 및 learning curve 필요

- Phantom.js

  - http://phannntomjs.org

  - chrome v8 엔진 기반의 테스팅 도구

  - javascript 기반

  - 페이지 비동기 호출 체크 가능

  - 단점 : 멀티 브라우징 테스트 안 됨. 최근 realease 배포 없음

# 배포 자동화

- 연결 유지

- router 재기동 정책

- load balnacer 설정 자동화

- 배포 실패시 rollback 처리

- martin fowler가 2010 년 blud green deploy 방식 정의

- green zone와 blue zone을 반영할 때마 스위치하는 방식

- 배포 문제 발생시 이전 서비스로 스위칭함

## Zero time deployment

- deploy된 위치가 아닌 다른 영역에 배포후 domain이 바라보는 곳으로 스위칭(AWS Beanstalk, pivotal cloude)

- I7 layer 등을 재기동하지 않고 config을 reload 하여 사용

- 배포된 서버로 연결을 switching하는 방식

- 잦는 switching은 장애 발생 가능성 있음

## AWS CodePipeline

- 배포를 리전별 그룹별 단계별로 배포 가능

- 서버 배포시 여러 형태의 배포 설정 지원

# Fail Over

- High Availability 구성 필수

- I4, I7 영역에서 rise & fall option 설정 가능(ha proxy)

- url health check하는 방식

## Circuit Breaker

- Martin Fowler 정의

- 특정 연결 장애 지속시 연결을 끊고 다른 대체 method를 호출.

- 특정 연결이 정상시 다시 원래 method 호출

- Netflix의 Hystrix(Spring cloud netflix)나 FailSafe, javaslang-circuit breaker 등

# Scale Out

## 같은서버 적용시

- 같은 서버에 적용시 port나 로그 등 중복 가능성

- session clustering 설정 필요(redis나 gemfire 같은 memory db 고려)

- nodejs는 pm2만 설치하면 가능

### Docker-compose

- https://www.docker.com/products/docker-compose

- docker image를 yml 설정파일로 관리

- 여러 개의 docker image를 docker-compose up 명령으로 여러 서버 기동 및 부분 기동 가능

- docker는 container 안에서 독립적으로 port나 로그 경로 중복 가능

## 다른 서버 적용시

- vm 기반 cloud 환경 추천

- on premise 환경일 경우 ansible, docker-swarm,kubernetes 로 관리

- vm 이미지 기반일 경우 vagrant 로 관리

## 기타

- Apache Mesos로 자원 관리

- Marathon으로 was 및 job 관리

- Marathon 은 auto recovery, scale change, load balance, health check 등을 UI 로 관리

# 기타

- Slack을 사용하여 채팅 명령어로 배포하는 Hubot, Deploy Bot 서비스

- AWS 에서는 Alexa Skills Kit 음성서비스로 관리
