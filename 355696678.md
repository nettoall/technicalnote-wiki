## 1.1 Repository를 이용해서 설치

인터넷이 가능한 환경에서 지원

### 1.1.1 ubuntu 설치

1.  sudo apt-get update

2.  필수 프로그램 설치

    - apt-transport-https (deprecated)

    - ca-certificates

    - curl

    - gnupg-agent (deprecated)

    - software-properties-common(deprecated)

    - gnupg, lsb-release (최신)\

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      $ sudo apt-get update
      $ sudo apt-get install \
          ca-certificates \
          curl \
          gnupg \
          lsb-release
      ```

      </div>

      </div>

3.  GPG key 저장\

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    $ sudo mkdir -p /etc/apt/keyrings
    $ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
    ```

    </div>

    </div>

4.  repository 설정\

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    $  echo \
      "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
      $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
    ```

    </div>

    </div>

5.  docker engine 설치\

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    $ sudo apt-get update
    $ sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin
    ```

    </div>

    </div>

6.  docker 설치 확인\

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    $ sudo docker run hello-world
    $ sudo docker version # docker container 설치 안 하고 docker engine 설치 여부 확인 
    ```

    </div>

    </div>

    1.  hello-world 관련 docker container가 없으면 가져와서 설치 후 실행함

### 1.1.2 centos 설치

- centos 는 /etc/sudoers 파일에 guru 계정 추가 or root 설치 진행

- yum repository가 <a href="http://mirror.kakao.com" class="external-link" rel="nofollow">mirror.kakao.com</a> 으로 되어 있으나 docker 관련 정보를 못 가져옴 (아래 명령어 실행)\

    </div>

3.  docker 설치 확인\

4.  docker status 확인\

### 1.1.2 centos8 설치

- podman 또는 docker 중에서 선택해야 하는 것으로 추정됨

- docker를 사용하려면 dnf -y module disable container-tools 를 한 후, 다음 단계를 진행하는 것으로

5.  sudo dnf install docker-ce-cli --nobest -y

6.  systemctl status docker.service

7.  systemctl enable docker

8.  systemctl start docker

9.  dnf install curl -y

10. curl -L “https://github.com/docker/compose/release/downlaod/1.27.4`/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose`

11. chmod +x

# 2. 설정

## 2.1 docker 실행권한 주기

- docker 설치 후 실행권한은 root에만 있음.

- root 계정외에 다른 계정에서 다음 명령을 시험삼아 수행(Permission Denied 오류 발생)\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  docker ps
  ```

  </div>

  </div>

- docker가 설치되면 docker 그룹권한이 생성됨

- 일반 유저에게 docker 그룹 권한을 할당하여 실행권한을 줌 (일반 유저는 재로그인을 해야 할당된 권한 사용가능함)

<div class="code panel pdl" style="border-width: 1px;">

<div class="codeContent panelContent pdl">

``` syntaxhighlighter-pre
sudo usermod -a -G docker ${USER}
```

</div>

</div>

- 테스트 방법

  - sudo su - (루트 계정으로 접속)

  - su - \${USER} (원래 계정으로 로그인)

  - groups \${USER} (docker group에 들어갔는지 확인)

  - docker 실행

# 3. Docker-Compose

- Docker compose란, 여러 개의 컨테이너로부터 이루어진 서비스를 구축, 실행하는 순서를 자동으로 하여, 관리를 간단히하는 기능이다.

- docker-compose.yml 파일은 yaml으로 Docker 컨테이너에 관한 실행 옵션(build 옵션도 포함되어 있는 경우도 있다)를 기재한 파일

## Docker-Compose 설치

- docker-compose 최신 release 버전 확인

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  curl -s https://api.github.com/repos/docker/compose/releases/latest | grep tag_name | cut -d '"' -f 4
  ```

  </div>

  </div>

- docker-compose 최신 버전 설치

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  sudo curl -sSL "https://github.com/docker/compose/releases/download/$(curl -s https://api.github.com/repos/docker/compose/releases/latest | grep tag_name | cut -d '"' -f 4)/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
  ```

  </div>

  </div>

- docker-compose 버전 지정해서 설치

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  sudo curl -sSL "https://github.com/docker/compose/releases/download/v2.18.1/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
  ```

  </div>

  </div>

- 권한주기

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  chmod +x /usr/local/bin/docker-compose
  ```

  </div>

  </div>

- docker-compose vserion 정보 조회

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  docker-compose --version
  ```

  </div>

  </div>

- docker 실행 확인

  - docker 엔진을 시스템에서 자동 실행하도록 하는 방법(systemctl)과 수동으로 실행하는 방법이 존재

  - wsl 에서는 루트 시스템 프로세스가 systemd가 아닌 init 이기 때문에 서비스가 떠 있지 않음

  - wsl 버전이 업그레이드 되면서 systemd 체계로 바꿀 수 있음(아마도 초기 설정이 init 이기 때문에 수동으로 systemd로 바꿔야 하는 것으로 추정됨)

  - 변경하기전 pstree 명령으로 조회

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    init(Ubuntu-22.─┬─SessionLeader───Relay(10)─┬─zsh───pstree
                    │                           ├─zsh───gitstatusd-linu───16*[{gitstatusd-linu}]
                    │                           └─2*[zsh]
                    ├─init───{init}
                    └─{init(Ubuntu-22.}
    ```

    </div>

    </div>

- docker 실행(수동으로 실행방법)

  - service docker status (docker 상태 확인)

  - sudo service docker start

- systemctl 로 docker 실행 방법은 “WSL의 루트 시스템 프로세스 바꾸기” 참고

## WSL의 루트 시스템 프로세스 바꾸기

- 기존 init에서 systemd로 변경

1.  /etc 밑에 wsl.conf 파일 생성

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    $ sudo vim wsl.conf
    ```

    </div>

    </div>

2.  아래 내용으로 파일 내용 입력

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    [boot]
    systemd=true
    ```

    </div>

    </div>

3.  wsl 창을 닫고 CMD나 powershell 에서 다음 명령어 실행(루트 시스템 프로세스를 변경하기 위해 필수)

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    wsl.exe --shutdown
    ```

    </div>

    </div>

4.  wsl 접속 후 pstree 로 확인

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    systemd─┬─2*[agetty]
            ├─containerd───15*[{containerd}]
            ├─containerd-shim─┬─mariadbd───12*[{mariadbd}]
            │                 └─10*[{containerd-shim}]
            ├─cron
            ├─dbus-daemon
            ├─dockerd─┬─2*[docker-proxy───7*[{docker-proxy}]]
            │         └─12*[{dockerd}]
            ├─init-systemd(Ub─┬─SessionLeader───Relay(769)─┬─zsh───pstree
            │                 │                            ├─zsh───gitstatusd-linu───16*[{gitstatusd-linu}]
            │                 │                            └─2*[zsh]
            │                 ├─init───{init}
            │                 ├─login───zsh
            │                 └─{init-systemd(Ub}
            ├─networkd-dispat
            ├─rsyslogd───3*[{rsyslogd}]
            ├─snapd───15*[{snapd}]
            ├─5*[snapfuse]
            ├─subiquity-serve───python3.8
            ├─systemd───(sd-pam)
            ├─systemd-journal
            ├─systemd-logind
            ├─systemd-resolve
            ├─systemd-udevd
            └─unattended-upgr───{unattended-upgr}
    ```

    </div>

    </div>

5.  sudo systemctl status docker 로 docker 실행 여부 확인 가능(systemctl 로 docker 실행 가능)

    - sudo systemctl start docker

    - sudo systemctl enable docker

    - systemctl은 시스템이 시작하면 계속 서비스를 떠 있게 함

6.  docker 실행 확인

    - service docker status
